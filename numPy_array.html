Working with the numPy Arrays

What is numPy?

Well, the answer to that question is that numPy is a library available Python. It comes in very handy when working with large data sets. We'll see some of the examples about how to use this library how to create arrays using this library. NumPy library is a multi-dimentional Array library. Using this library we can create multi-dimentional arrays like 1D, 2D, 3D, 4D and so on. The numPy arrays store homogeneous(data of same data type) data.

Why numPy library when we have lists?

It is correct that we can use lists to create arrays but there are some limitations to it. Let's see why we use numPy over lists :
  • First thing why we use this library is that it takes way less space than a list and hence it is Faster.
    The space required by an integer stored using numPy is either 32Int by default or we can decrease this size to 16Int or 8Int if we want which we'll see when we start creating numPy arrays. But the space taken by a list integer is quite big as you can see in the picture above. It requires 8 bytes for Reference Count, 8 bytes for Object Type, 8 bytes for Object Value, and around 4 bytes for Size. Using numPy array uses less memory.

  • The other reason why it is better to use numPy is that lists while iterating do the type checking but since numPy arrays contain homogeneous(same type) data, there is no type checking.

  • numPy utilizes contiguous memory and list doesn't. Hence, even the cache is utilized vey efficiently in this case.

  • Another reason of using numPy arrays is that we can do operations like addition, subtraction, multiplication, and divsion on numPy arrays without much difficulty unlike in lists.
